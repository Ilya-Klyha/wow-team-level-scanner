================================================================================
TEAM DoR
================================================================================

Criterion: Task description
Description: Every Task should be well described:
- What has to be done
- Has acceptance criteria and should be related to Epic Acceptance Criteria
- UX design (if applicable) should be represent actual workflow and in form of
  Figma (link)
- Every Tasks should be connected to Epic
- Dependencies (if applicable) in description
- Dependences/Relationships between tasks should be visible in Jira (if
  applicable)

Criterion: Estimates Provided
Description: The development team has provided an estimate for the work, or the
ticket has been broken down to a size that allows for reasonable estimation.

Criterion: Work size
Description: Scope is small enough to fit within a sprint

Additional Team Agreements:
- Independent Scrum team with independent ceremonies
- If task has "In Progress" status more than 5 days, Jira comment required
  explaining current work status (blockers, dependencies, etc.)
- Working agreement with UX/UI team (Agata Pietralik)
- Ad reporting cooperation with team Bigos (Data team)
- Dedicated architect: Siarhei Ihnatsenka
- Scrum Master rotation every 2 iterations
- Scrum Master responsible for updating Team Working Agreement
