Issue: EEEW-333
Type: Task
Status: In Progress
Priority: High
Summary: Fix Kyverno policy violation on ingress-nginx-dsp-bidding-systems admission job
Assignee: Serhiy Danchenko
URL: https://adgear.atlassian.net/browse/EEEW-333
Created: 2026-05-15T10:58:24.206-0400
Updated: 2026-05-20T14:07:57.218-0400
================================================================================

Description:
## Problem

The ingress-nginx admission webhook job in namespace `op-ingress-nginx-dsp-bidding-systems` is blocked by Kyverno policy `require-requests-limits`:

```
one or more objects failed to apply, reason: admission webhook "validate.kyverno.svc-fail" denied the request:

resource Job/op-ingress-nginx-dsp-bidding-systems/ingress-nginx-dsp-bidding-systems-admission-create was blocked due to the following policies

require-requests-limits:
  autogen-validate-resources: 'validation error: - CPU and Memory resource requests
    are required. - Memory resource limit is required.'
```

## Root Cause

The admission-create Job template in the ingress-nginx Helm chart does not have resource requests/limits configured, which violates the cluster Kyverno policy.

## Fix

Add resource requests and limits to the admission webhook job in the ingress-nginx Helm values for the `dsp-bidding-systems` deployment. The relevant container at `/spec/template/spec/containers/0/resources/limits/` needs:

* CPU request
* Memory request
* Memory limit

## Reference

* Policy docs: https://adgear.atlassian.net/wiki/spaces/ENG/pages/20158808082/Require+requests+limits
